# Devnet MVP PRD traceability

This is the reviewed traceability audit required by `architecture.md` §22. It covers the Devnet MVP only. A local pass means the repository implementation and a direct local test exist; it is not a substitute for deployment evidence. “External gate” means the implementation seam exists but the named credential, device, infrastructure, or public Devnet proof is unavailable. Mainnet-only PRD criteria are excluded by the user-approved scope and no Mainnet configuration is shipped.

## Acceptance-criterion disposition

The PRD §18 Devnet campaign, registration, activation, claims, accounting/refund, demo, and performance rows map to the tables above. The PRD’s Mainnet allowlist, separate Mainnet deployment, and Mainnet demo-rejection rows are not Devnet-MVP release criteria. This repository contains no Mainnet endpoint, program, authority, or mint configuration; promoting it to Mainnet would require a new architecture and review.

The complete local seam and the recorded public-Devnet program/config/mint/campaign/Instant-Demo registration, claim, transfer, faucet, overlap, completion, refund, vault closure, fixture release, fresh-reset, and authenticated TxLINE schema seams are green. Public release acceptance is deliberately **not** green until RB-1, RB-3, and RB-4 in `release-blockers.json` are resolved and the verified containers have a public runtime. Local results are not presented as evidence for those remaining external facts.
