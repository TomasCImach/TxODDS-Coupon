## External release gates

The core economic path is deployed on Devnet, but it cannot honestly be called a public release until these gates are completed. The requester confirmed the TxODDS public-data permission is solved; raw feeds and credentials remain server-only, public output is constrained to the approved derived display, and raw retention remains disabled unless explicitly covered.

1. Physical-device Passkeys coverage for message signing, transaction signing, recovery, export, and reconnect. The App ID is intentionally optional for Devnet and required only when `NEXT_PUBLIC_DEPLOYMENT_TIER=production`.
2. A target container registry/runtime to publish the verified images, managed PostgreSQL, required roles, immutable deployment digests, and public URL, then run clean-browser, rollback, and target-topology load/SSE/latency acceptance.
3. Independent smart-contract security, privacy/licensing, and promotion-law review appropriate for any public release beyond hackathon judging.

The nine architecture release blockers, accountable roles, and required evidence are machine-readable in [`release-blockers.json`](./release-blockers.json); seven are resolved and `pnpm release:check` reports RB-3 and RB-9. Exact user actions are in [`unblock-checklist.md`](./unblock-checklist.md). Requirement-level disposition is in [`prd-traceability.md`](./prd-traceability.md).

Until those gates close, the recorded Devnet Instant Demo seam and local-validator evidence are valid engineering proof, not public-release approval. No private key, JWT, licensed raw payload, Mainnet address, or valuable token is included in the repository.
