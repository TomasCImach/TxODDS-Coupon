- reward mint `DLuT…UAUK` is classic SPL, has six decimals, no freeze authority, and an operator-controlled mint authority;
- untouched judge campaign `33cy…v2mT` passed create, exact 15-token funding, and activation, with all three immutable five-winner/one-token rounds unused and projected into PostgreSQL; registration remains open through July 17 at 15:18:37 UTC;

- the completed campaign `8bek…Ea4y` then passed oracle completion, permissionless refundable transition, fixed-destination 14-token residual refund, vault closure, and fixture-slot release; direct account decoding proved `15,000,000 funded = 1,000,000 paid + 14,000,000 refunded`, with both closed accounts absent;
- on acceptance campaign `DCii…Qvu8`, a fresh in-memory, zero-SOL fan created Registration PDA `9N3q…TnpZ`, received durable sequence 1, and settled Claim PDA `j2aN…bDpV` at rank 1 for exactly 1,000,000 base units into a newly created classic ATA;
- that fan then explicitly signed a platform-fee-paid classic SPL transfer of the full reward to a fresh Devnet destination; direct readback showed source balance zero and destination balance exactly 1,000,000 base units;
- a fresh sponsor wallet signed a one-time faucet transaction that created its classic ATA and received exactly 500,000,000 base units of the valueless mint; a duplicate build was rejected with HTTP 409;
- the fresh acceptance round closed with one winner and direct campaign decoding proved `15,000,000 funded = 1,000,000 paid + 14,000,000 vault`; and
- request logging now redacts capability-bearing URLs, while an invalid durable command exhausted a 12-attempt budget, moved to the audited dead-letter set, and stopped retrying.

The deployment-specific 0.5 SOL circuit-breaker floor is limited to this 15-winner demo. The observed fee-payer balance after the completed lifecycle, acceptance campaign, registration, claim, transfer, round close, untouched judge campaign, and faucet acceptance was 0.74445817 SOL; the conservative full-demo rent estimate was 0.104922 SOL. Current rent makes 100 fresh Registration/Claim/ATA sets plus fees approximately 0.611892 SOL, so preserving the floor requires at least a 0.36743383 SOL top-up; 1 Devnet SOL is recommended for retry headroom.

The requester selected the Foundation public RPC for the Devnet MVP. It carried all targeted acceptance writes and reads. Its broad `getProgramAccounts` path returned HTTP 429, so the indexer remains alive, keeps live-log ingestion, and falls back to owner/length-validated reads of the configured demo campaign. The unresolved 100-winner contention/drain proof remains RB-4 rather than being disguised as an infrastructure pass.

Generated validator ledgers, program keypairs, test wallets, browser traces, screenshots, build output, and environment files are ignored and are not release artifacts.

## External release gates

The core economic path is deployed on Devnet, but it cannot honestly be called a public release until these gates are completed:

1. Written TxLINE approval for the minimal public derived display/caching policy; live terminal/correction/schema validation is complete.
2. A Passkeys production App ID plus physical-device coverage for message signing, transaction signing, recovery, and export.
3. A fee-payer top-up and deployed 100-winner contention/drain/retry benchmark; public RPC remains the approved MVP endpoint.
4. A target container registry/runtime to publish the verified images and record immutable deployment digests and the public URL.
5. Independent smart-contract security, privacy/licensing, and promotion-law review appropriate for any public release beyond hackathon judging.

The eight architecture release blockers, accountable roles, and required evidence are machine-readable in [`release-blockers.json`](./release-blockers.json); five are resolved and `pnpm release:check` now reports only RB-1, RB-3, and RB-4. Exact user actions are in [`unblock-checklist.md`](./unblock-checklist.md). Requirement-level disposition is in [`prd-traceability.md`](./prd-traceability.md).

Until those gates close, the recorded Devnet Instant Demo seam and local-validator evidence are valid engineering proof, not public-release approval. No private key, JWT, licensed raw payload, Mainnet address, or valuable token is included in the repository.
