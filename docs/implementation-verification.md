## Public Devnet evidence

The non-secret deployment ledger is [`devnet-deployment.json`](./devnet-deployment.json). Direct readback on July 16 proved:

- program `2NUW…xZnc` is executable under the upgradeable loader and its 602,064 deployed bytes hash exactly to `38d357…4d33`;
- `PlatformConfig` `9FiS…pWKj` has the intended distinct admin, oracle, relayer, and demo authorities, Devnet domain, reward mint, and zero pause mask;
- reward mint `DLuT…UAUK` is classic SPL, has six decimals, no freeze authority, and an operator-controlled mint authority;
- untouched judge campaign `33cy…v2mT` passed create, exact 15-token funding, and activation, with all three immutable five-winner/one-token rounds unused and projected into PostgreSQL; registration remains open through July 17 at 15:18:37 UTC;
- the API started against the on-chain config and PostgreSQL state, returned healthy public/internal checks, exposed open fee-payer admission, and returned the campaign through its public endpoint;
- two visibly synthetic commands opened independent Round PDAs `24g5…2vSy` and `H1v9…AoEa` at the same time, after which the lifecycle worker closed both with confirmed transactions;
- the completed campaign `8bek…Ea4y` then passed oracle completion, permissionless refundable transition, fixed-destination 14-token residual refund, vault closure, and fixture-slot release; direct account decoding proved `15,000,000 funded = 1,000,000 paid + 14,000,000 refunded`, with both closed accounts absent;
- on acceptance campaign `DCii…Qvu8`, a fresh in-memory, zero-SOL fan created Registration PDA `9N3q…TnpZ`, received durable sequence 1, and settled Claim PDA `j2aN…bDpV` at rank 1 for exactly 1,000,000 base units into a newly created classic ATA;
- that fan then explicitly signed a platform-fee-paid classic SPL transfer of the full reward to a fresh Devnet destination; direct readback showed source balance zero and destination balance exactly 1,000,000 base units;
- the fresh acceptance round closed with one winner and direct campaign decoding proved `15,000,000 funded = 1,000,000 paid + 14,000,000 vault`; and
- request logging now redacts capability-bearing URLs, while an invalid durable command exhausted a 12-attempt budget, moved to the audited dead-letter set, and stopped retrying.

The deployment-specific 0.5 SOL circuit-breaker floor is limited to this 15-winner demo. The observed fee-payer balance after the completed lifecycle, acceptance campaign, registration, claim, transfer, round close, and untouched judge campaign was 0.74650767 SOL; the conservative full-demo rent estimate was 0.104922 SOL. A larger campaign or the full 1,000-registration/800-claim capacity target still requires a separately funded fee payer and staged capacity proof.

The Foundation public RPC carried all targeted acceptance writes and reads. Its broad `getProgramAccounts` path returned HTTP 429, so the indexer now remains alive, keeps live-log ingestion, and falls back to owner/length-validated reads of the configured demo campaign. A managed RPC is still required for release-scale discovery, history, and burst acceptance.

Generated validator ledgers, program keypairs, test wallets, browser traces, screenshots, build output, and environment files are ignored and are not release artifacts.

## External release gates

The core economic path is deployed on Devnet, but it cannot honestly be called a public release until these gates are completed:

1. Written TxLINE approval for public derived display/retention plus authenticated terminal/correction samples; the current credentials already returned HTTP 200 and seven schema-valid Devnet fixtures.
2. A managed Devnet RPC/WebSocket provider sized for the burst; public endpoints were sufficient for the proven seam but rate-limited program upload and one indexer attempt.
3. A Passkeys app ID plus physical-device coverage for message signing, transaction signing, recovery, export, and judge Quickstart containment.
4. Public faucet/labeling review for the deployed valueless mint and a funded capacity plan above the 15-winner judge campaign.
5. A target container registry/runtime to publish the verified images and record immutable deployment digests, followed by external-wallet, passkey, managed-RPC burst, and browser/device acceptance in [`docs/runbooks/devnet-release.md`](./runbooks/devnet-release.md). Transfer, overlap, completion/refund, and fixture release already have direct public-Devnet evidence.
6. Independent smart-contract security, privacy/licensing, and promotion-law review appropriate for any public release.

The eight architecture release blockers, accountable roles, and required evidence are machine-readable in [`release-blockers.json`](./release-blockers.json); `pnpm release:check` fails until every row is resolved. Requirement-level disposition is in [`prd-traceability.md`](./prd-traceability.md).

Until those gates close, the recorded Devnet Instant Demo seam and local-validator evidence are valid engineering proof, not public-release approval. No private key, JWT, licensed raw payload, Mainnet address, or valuable token is included in the repository.
